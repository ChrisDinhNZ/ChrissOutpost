Create and publish an Angular library to NPM package registry

Creating an Angular library is a great way to break an application's logic into modules with specific functionalities. It is also less coupled with the applications that uses it therefore provides better code reuse and software maintainability.

So why would I want to publish an Angular library to the NPM package registry? Well I have been tinkering with a messaging protocol so that I can communicate between embedded devices and the web. With the web, I anticipate multiple web applications interacting with each other so having a published package in one place just makes everything much simpler. For web applications I leaned more towards Angular as my go to SPA framework to use.
