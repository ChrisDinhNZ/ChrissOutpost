Create and publish an Angular library to the NPM package registry

Creating an Angular library is a great way to break an application's logic into modules with specific functionalities. It is also less coupled with the applications that consumes it therefore provides better code reuse and software maintainability.

So why do I want to publish an Angular library to the NPM package registry? Well I have been tinkering with a messaging protocol so that I can communicate between embedded devices and the web. I anticipate having multiple web applications interacting with each other so having a published package in one place just makes my dev flow much simpler and less error prone.

In this blog I will work through the following:

  • Create library
  • Publish library
  • Import and use library
Create library

To create this library I will need Angular CLI, which I have already installed.

The following command will create a "workspace" for the library. The --create-application=false flag tells the CLI not to scaffold the default Angular application since I am going to create a library.

ng new SMoS-npm --create-application=false

The previous command will also initialise a git repository, which is good because I preferably have my work version controlled. I am just going to go ahead and push what I have so far to GitHub (note that not everything under the SMoS-npm folder will be push to the remote repository as files and folders defined in .gitignore will not be included).

cd SMoS-npm
git remote add origin https://github.com/ChrisDinhNZ/SMoS-npm.git
git branch -M main
git push -u origin main

The following command creates the projects/smos-js folder, which contains a component and a service inside an NgModule.

ng generate library smos-js

Since I only intends for the library to provide a service, I can get rid of smos-js.component.ts and any reference to it.
